定位收斂

不是放棄 IM，是先找到更窄的進場理由

原方向仍成立，但首頁需要先回答「使用者為什麼今天要進來」。

Product judgment

通用 IM 定位太寬

V1 不應同時對標 WhatsApp、Telegram、Discord、淘寶、Amazon、小紅書和 TikTok。那會讓首頁像功能清單，而不是一個清楚的進場場景。

Recommended pattern

先從有組織者的空間開始

創作者、商家、教育者、專案團隊和私密組織本來就有聚集人群的理由；IM 變成承接內容、討論、服務和長期關係的空間。

V1 architecture implication

仍保留原 IM 底座

私聊、群聊、好友、搜尋、通知、成員身份、權限、檢舉、封鎖、文件、公告和群管理都繼續成立，只是首個 landing 場景更具體。

需求場景

第一批操作者本來就有聚集人群的理由

使用者不是為了下載另一個 IM 而來，而是為了跟隨、學習、提問、參與活動或獲得持續服務。

01

Creator / 頻道主社群

高信任興趣消費，例如茶、酒、收藏、戶外裝備、攝影器材。

需要內容種草、專家解釋、社群討論和商品轉化。

用途：付費社群、課程、會員討論、直播活動、私聊、訂閱、長期關係管理。
02

教育 / 課程 / 訓練營

金融、投資教育社群、高客單價知識產品、課程或訓練營。

使用者購買前需要信任、問答、社群氛圍和老師人設。

用途：課程交付、課後討論、作業、Q&A、班級群、office hour、進度追蹤。
03

商業 / 專業服務社群

顧問、品牌、門店、專業服務、小型商家與高信任銷售。

關鍵不是一次成交，而是持續教育、回覆問題和關係轉化。

用途：內容教育、客戶提問、諮詢、活動公告、會員服務、關係轉化。
04

私密組織 / 專案團隊

公司、專案、協會、俱樂部、投資圈、校友會或私域組織。

需要公告、文件、成員權限、討論分流和一對一溝通。

用途：成員管理、資源庫、活動節奏、角色權限、專案討論、私密頻道。
競品缺口

現有工具各有強項，但不等於這個 landing 場景

新方向不是複製某一個競品，而是把頻道、社群、私聊、會員和治理放進同一個 IM 容器。

WhatsApp

熟人溝通強，商業社群與內容承接弱。

Telegram

公開頻道強，但私聊、群、頻道容易混在一起，商業關係難管理。

Discord

社群治理強，但結構較重，大眾用戶學習成本高。

傳統客服系統

工單與服務管理強，但缺少社群、內容與自然關係沉澱。

核心機制

把公開內容、社群討論與私聊轉化接成同一條路

核心不是「多一個社群頁」，而是讓每次互動都帶著上下文回到 IM 關係。

場景 A

使用者發現一個商家

外部內容 / 廣告 / QR 公開頻道頁：先看內容，建立信任 討論群 / Topic：按問題、活動、服務、資源分流 私聊 / 商業對話：帶著上下文進入一對一溝通 服務記錄 / 標籤 / 復購觸達：關係持續沉澱
場景 B

商家經營一個使用者池

發布內容 外部連結導流 頻道承接 群組分流討論 私聊轉化 標籤化營運 後續觸達
01

頻道負責低噪音內容承接

公告、內容、活動、商品資訊不被聊天淹沒。

02

把大群拆成可理解的問題空間

問答、活動、資源、討論分開，降低新成員進入成本。

03

商業對話帶著上下文進入私聊

使用者從哪個內容、群組和問題進來，商家可以看見。

04

信任與權限貫穿全流程

公開入口、群組、商家身份、客服、AI、API 都要被治理。

會員在同一個空間內可以做什麼

內容、關係與長期互動不要分散在多個產品

  • 閱讀官方公告
  • 加入主題討論
  • 取得課程或資源
  • 與管理員、創作者、導師或成員聊天
  • 加入群組或私密頻道
  • 搜尋成員並加為聯絡人
  • 獲得積分、等級、獎勵或紅包式激勵
  • 在同一產品內建立長期關係

完整模組細節

13 個模組形成一個社群優先的 IM 架構

模組不是一次全部做滿，而是讓 V1 的資料模型和產品語言能支撐後續擴展。

Module 1

Community Space

Purpose：創作者、商家、團隊、公司、課程、專案或私密社群的頂層容器。

  • 空間名稱、Logo、封面
  • 公開或私密空間
  • 創作者 / 擁有者檔案
  • About page
  • 成員數與活動狀態
  • Home、Community、Classroom、Chat、Events、Members、Resources
  • owner、admin、moderator、member、guest、paid member、VIP、student、employee
Module 2

Public Landing / 外部入口

Purpose：讓使用者在註冊或下載前先預覽價值。

  • 公開空間頁
  • 創作者 / 組織介紹
  • 免費內容預覽
  • 熱門貼文預覽
  • 課程或會員預覽
  • 成員見證
  • Join CTA、邀請連結、推薦連結、QR code
  • UTM / campaign source tracking
Module 3

Identity & Member System

Purpose：保留原 IM 底座，同時支撐社群成員身份。

  • 手機、email、社交登入
  • Username / ID 搜尋
  • 公開個人檔案
  • 每個空間內的成員檔案
  • 好友 / 聯絡人關係
  • follow / join 關係
  • 封鎖、檢舉、線上狀態
  • 成員標籤、角色與權限身份
Module 4

Membership & Access

Purpose：控制誰能看見什麼、加入什麼、解鎖什麼。

  • 免費社群、私密社群、邀請制社群
  • 付費會員、試用權限、VIP tier
  • 課程 access、cohort access
  • Role-based access
  • 課程、聊天室、資源、活動解鎖
Module 5

Community Feed / 社區討論

Purpose：讓有價值的討論可以被搜尋和沉澱，而不是被群聊速度沖走。

  • 貼文、問題、投票、公告、資源
  • wins / progress updates
  • 留言、回覆、反應
  • 最佳答案、分類、標籤
  • 置頂、收藏、檢舉、管理
  • 依最新、熱門、未回答排序
Module 6

Classroom / 課程與資料

Purpose：交付課程、課時、文件與結構化學習內容。

  • 課程、模組、課時
  • 影片、文字、文件、連結
  • 測驗、作業、完成狀態
  • 進度追蹤、課時留言
  • 關聯討論 topic
  • 依會員或等級解鎖
  • Resource library
Module 7

IM & Chat

Purpose：保留核心 IM 能力，形成比論壇型產品更強的關係優勢。

  • 1:1 私聊、成員互聊、創作者 / 管理員對成員聊天
  • 群聊、cohort chat、VIP chat、event chat、course discussion chat
  • 文字、語音、圖片、文件、連結預覽
  • 回覆、反應、已讀、置頂、通知
  • Contextual DM：來源內容、加入課程、提問、會員狀態、近期活動
Module 8

Events / Calendar

Purpose：在社群內建立節奏和週期性參與。

  • 直播課、AMA、office hour、webinar
  • Challenge、meetup、event calendar
  • RSVP、提醒、live chat
  • 回放、出席追蹤、活動後討論
Module 9

Incentives / 積分、紅包、等級、裂變

Purpose：連接熟悉的紅包、積分、獎勵、裂變和參與激勵能力。

  • 積分、等級、徽章、排行榜
  • 每日簽到、課程完成獎勵、有幫助回答獎勵
  • 邀請獎勵、推薦獎勵、紅包活動
  • 群組活動獎勵、活動參與獎勵
  • 依等級解鎖內容或聊天權限
  • 反垃圾規則，避免低品質刷分
Module 10

Creator / Admin Console

Purpose：讓操作者把社群當成生意或組織來經營。

  • 成員列表、成員活動、成員標籤
  • 批准、移除、封禁成員
  • 角色分配、內容管理、課程管理
  • 活動管理、公告發布、聊天管理
  • 活動分析、課程完成分析、member journey
  • 收入 / 訂閱總覽、campaign source tracking
Module 11

Trust & Governance

Purpose：讓社群成長後仍安全、可管理。

  • 檢舉、封鎖、靜音、封禁
  • 管理員、版主、加入審核
  • 貼文管理、留言管理
  • 垃圾訊息偵測、連結安全
  • 支付爭議紀錄
  • 管理員操作、內容移除、權限變更紀錄
Module 12

Search & Discovery

Purpose：幫使用者找到人、內容、群組、課程、活動和空間。

  • 搜尋 user ID、成員、貼文、課程
  • 搜尋文件、活動、群組、公開空間
  • 依標籤、角色、課程、topic 篩選
  • 公開發現可後續加入，不改核心架構
Module 13

Payments / Subscription

Purpose：支撐創作者變現和付費社群，但不變成標準電商平台。

  • 月訂閱、年訂閱、一次性課程付款
  • 付費 cohort、VIP 會員、coupon、trial
  • 退款狀態、付款紀錄、訂閱狀態
  • 與 access control 整合
