V1 核心閉環

只做能讓閉環成立的模組

判斷原則：不能支撐「入口 → 討論 → 回訪」的功能，不進 V1。

01

外部入口

先看得到，再決定是否加入。

02

公開頻道預覽

用 profile、貼文與 CTA 建立信任。

03

關注 / 加入

把瀏覽轉成可回訪的關係。

04

討論入口

從內容 CTA 進主題，而不是直接進噪音流。

05

主題互動

讓多人討論可找、可管、可回訪。

06

收件箱留存

用未讀、標籤與篩選維持秩序。

07

回訪

通知回到正確頻道、主題或聊天。

模組對閉環地圖

核心先做，支援只做到可用

V1 只按閉環必要性排序，不按功能想像排序。

核心閉環模組

必須進 V1，否則產品路線不成立

身份

登入、加入、私聊、治理。

頻道

公開入口與內容承接。

群組 / 主題

把內容轉成多人互動。

收件箱

未讀、私聊、回訪入口。

支援模組
搜尋 / 發現

V1 只做能找到頻道、群組與聊天的基本能力；推薦與 SEO 先不做。

信任 / 治理

V1 只做公開社群最低可用治理：管理員、檢舉、封鎖、靜音、移除。

通知

V1 只保證私聊、群組 / 主題、頻道更新能回到正確位置。

後續 / 擴展
創作者工具

進階排程、分析、多管理員放 Later；V1 先證明頻道到討論的閉環。

錢包 / 積分

激勵層會放大複雜度；除非核心社群閉環已被證明，否則不進 V1。

API / Bot

平台能力先保留方向，不作為第一版可交付範圍。

進階分析

不要用假儀表板製造確定性；V1 只需要基本事件紀錄支援驗證。

關鍵架構決策

五個需要核准的預設

每張卡只做一件事：核准、調整或否決。

決策 01 開啟互動 Demo

頻道內容如何導向討論？

預設：公開頻道預覽 → 討論 CTA → 主題優先群組。

核准作為 V1 預設。
調整加入有限留言預覽。
否決改純留言或純群組。

依據：Telegram 驗證頻道到討論；Discord / Forum 驗證主題分流。

風險：結構過重會增加加入摩擦。

決策 02 開啟互動 Demo

進入大型群組後第一屏看什麼？

預設：主題列表優先，露出輕量公告 / 規則。

核准第一屏用主題列表。
調整指定預設主題模板。
否決改訊息流或導覽優先。

依據：Topics / Forum 都用分區降低大群混雜。

風險：初版主題分類可能錯。

決策 03

收件箱如何組織私聊、群組與頻道？

預設：統一收件箱 + 類型標籤 + 篩選。

核准維持一個主收件箱。
調整加 Chats / Channels 分頁。
否決改完全分離 Updates。

依據：IM 關係主要回到聊天列表。

風險：頻道更新可能壓過私人聊天。

決策 04

登入 / 加入前公開什麼？

預設：公開頻道 profile + 精選貼文；討論需加入。

核准公開 profile 與精選貼文。
調整開放少量討論摘要。
否決改無預覽或全公開。

依據：預覽降低進入門檻；討論保留加入邊界。

風險：公開內容規則要先定清楚。

決策 05

Day one 需要哪些治理？

預設：管理員 + 檢舉 / 封鎖 / 靜音 + 移除成員。

核准Day one 納入基本治理。
調整延後 log 或 topic 控制。
否決改最小管理或完整系統。

依據：公開社群需要最低安全邊界。

風險：太重拖慢 V1，太輕不可用。

建議建置順序

照依賴順序建置

先底座，再入口、討論、回訪、治理。

Phase 1

通訊底座

Identity、inbox shell、private chat、message model。

Phase 2

公開入口

Public channel page、channel profile、post list、shareable URL。

Phase 3

社群討論

Group、topic list、topic message flow、discussion CTA。

Phase 4

回訪閉環

Unread、notifications、mute、deep link back to channel/topic/chat。

Phase 5

最低治理

Admin、report、block、mute、remove member、moderation log。

剩餘假設

只追四個會改路線的假設

用實測驗證，不在會議中猜答案。

假設 01

公開頻道預覽足以提高加入意願

驗證：2-3 個真實頻道測公開頁 → join / follow。

若為假：縮小公開預覽，強化邀請導入。

假設 02

主題列表能降低大群進入成本

驗證：比較訊息流 first 與主題 first。

若為假：公告 + 熱門訊息流 first。

假設 03

統一收件箱不會被頻道更新污染

驗證：測類型標籤、篩選、預設靜音。

若為假：改 Chats / Channels 分頁。

假設 04

基本治理足以支撐第一批公開社群

驗證：小社群測檢舉、封鎖、移除。

若為假：提前加入角色權限或審核。
