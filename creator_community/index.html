空間型 IM

以空間為核心的內容、訂閱與即時互動平台

定位收斂

為什麼要從泛 IM 收窄到空間主與高信任關係？

問題不是 IM 能做什麼，而是誰有足夠強的理由，把用戶帶進一個新的 IM-based 空間。

不能假設 01

用戶會來這裡「買東西」

有明確購物需求時，用戶會去 Amazon、淘寶、Shopee；沒有購物需求時，需要靠內容、信任、空間氛圍和他人經驗慢慢轉化。

更適合：高信任、高解釋成本、高互動成本、高復購或長週期關係的產品與服務。
不能假設 02

用戶會把它當成一般 IM 替代品

用戶已經有 WhatsApp、Telegram、微信、LINE、Discord。新產品若只提供私聊、群聊、通知和好友，沒有足夠遷移理由。

第一個問題應該是：哪一類人能把用戶帶進新的 IM-based 空間？
不能假設 03

早期用戶會為平台而來

早期用戶更可能為某個空間主、課程、討論區、活動、服務、福利、資料或組織而來，而不是因為「這是一個新 IM」。

所以第一版應服務能帶來用戶的空間主：老師、專業服務負責人、內容型空間主、私密組織與項目負責人。
看到內容 / 推薦 / 活動 / 朋友分享 有興趣，但還不夠信任 想看更多內容與他人提問 提出具體問題 信任建立後購買、報名、預約或留下
產品解決的不是「讓用戶買東西」。

它幫助用戶在決策前完成理解、觀察、提問與信任建立；也幫助空間主把一次性內容觸達，轉化成可互動、可服務、可付費、可留存的長期關係。

需求場景

第一批空間主本來就有聚集人群的理由

使用者不是為了下載另一個 IM 而來，而是為了跟隨、學習、提問、參與活動或獲得持續服務。

01

空間主 / 高信任興趣空間

高信任興趣消費，例如茶、酒、收藏、戶外裝備、攝影器材。

需要內容種草、專家解釋、討論區互動和商品轉化。

用途：付費空間、課程、會員討論、直播活動、私聊、訂閱、長期關係管理。
02

教育 / 課程 / 訓練營

金融、投資教育空間、高客單價知識產品、課程或訓練營。

使用者購買前需要信任、問答、空間氛圍和老師人設。

用途：課程交付、課後討論、作業、Q&A、班級群、office hour、進度追蹤。
03

專業服務 / 內容型空間

顧問、品牌、門店、專業服務、小型內容型空間主與高信任銷售。

關鍵不是一次成交，而是持續教育、回覆問題和關係轉化。

用途：內容教育、客戶提問、諮詢、活動公告、會員服務、關係轉化。
04

私密組織 / 專案團隊

公司、專案、協會、俱樂部、投資圈、校友會或私域組織。

需要公告、文件、成員權限、討論分流和一對一溝通。

用途：成員管理、資源庫、活動節奏、角色權限、專案討論、私密頻道。
競品缺口

現有工具各有強項，但不等於這個 landing 場景

新方向不是複製某一個競品，而是把內容與公告、討論區、聊天、會員和治理放進同一個空間型 IM 容器。

WhatsApp

熟人溝通強，空間化內容承接與成員關係管理弱。

Telegram

公開頻道強，但私聊、群、頻道容易混在一起，商業關係難管理。

Discord

空間治理強，但結構較重，大眾用戶學習成本高。

傳統客服系統

工單與服務管理強，但缺少空間、內容與自然關係沉澱。

核心機制

空間型 IM 要把入口、內容、討論、交付和聊天接成閉環

重點不是立即成交，也不是替代所有 IM，而是讓空間主把一次觸達變成可持續經營的成員關係。

場景 A

成員從外部內容進入空間

Public Entry：內容、廣告、QR、朋友分享 公開承接頁：先理解空間主、內容和會員價值 空間：進入公告、討論區、交付中心或活動 Chat：帶著來源、問題和會員狀態進入對話 通知與權限：把成員帶回原內容、討論或交付項目
場景 B

空間主經營一個長期成員池

內容與公告 討論區分流 交付中心沉澱資料 活動建立節奏 Chat 承接高意圖問題 經營後台管理成員 通知回訪
01

空間是核心容器

平台下面是空間；空間裡再組織群組、討論區、交付中心、活動、成員和聊天。

02

內容與公告負責低噪音承接

公告、內容、活動和服務更新不被群聊淹沒，先建立理解和信任。

03

討論區把問題拆開

問答、活動、服務、資源和經驗分享分開，降低新成員進入成本。

04

交付中心沉澱可交付價值

課程、文件、工具、案例、SOP、直播回放和會員資料都在交付中心內被權限控制。

05

Chat 承接高意圖關係

成員從哪個內容、討論、交付項目或活動進來，空間主能帶著上下文回覆。

06

信任與權限貫穿全流程

身份、成員、權限、付費狀態、檢舉、封鎖和審核紀錄要跨模組成立。

會員在同一個空間內可以做什麼

內容、關係與長期互動不要分散在多個產品

  • 閱讀官方公告
  • 加入主題討論
  • 取得交付中心內的課程、文件、工具或回放
  • 與空間主、管理員、協作者或成員聊天
  • 加入群組或私密討論區
  • 搜尋成員並加為聯絡人
  • 獲得積分、等級、獎勵或紅包式激勵
  • 在同一產品內建立長期關係

架構模組細節

17 個模組形成一個空間型 IM 架構

模組不是一次全部做滿，而是讓資料模型和產品語言能支撐後續擴展。

Module 1

空間 / Space

Purpose：空間主、團隊、公司、課程、專案或私密組織的頂層容器。

  • 空間名稱、Logo、封面
  • 公開或私密空間
  • 空間主 / Space Owner 檔案
  • About page
  • 成員數與活動狀態
  • Home、Discussion、Delivery Hub（含 Resources）、Chat、Events、Members
  • Space Owner、Admin、Moderator / Operator、Member、Guest、Paid Member、VIP
Module 2

外部入口 / Public Entry

必要

Purpose：把外部內容、廣告、QR、邀請連結和活動入口導入指定空間。

  • 邀請連結、QR code、分享連結
  • 外部內容 CTA、廣告入口
  • 入口來源與 campaign tracking
  • 導向公開承接頁、指定討論、交付內容或私聊
  • 風險檢查與入口權限
Module 3

Public Landing / 公開承接頁

Purpose：讓使用者在註冊或下載前先預覽價值。

  • 公開空間頁
  • 空間主 / 組織介紹
  • 免費內容預覽
  • 熱門貼文預覽
  • 課程或會員預覽
  • 成員見證
  • Join CTA、邀請連結、推薦連結、QR code
  • UTM / campaign source tracking
Module 4

身份與成員 / Identity & Members

Purpose：保留原 IM 底座，同時支撐空間成員身份。

  • 手機、email、社交登入
  • Username / ID 搜尋
  • 公開個人檔案
  • 每個空間內的成員檔案
  • 好友 / 聯絡人關係
  • follow / join 關係
  • 封鎖、檢舉、線上狀態
  • 成員標籤、角色與權限身份
Module 5

權限與會員 / Access & Membership

必要

Purpose：控制誰能看見什麼、加入什麼、解鎖什麼，並把付款 / 訂閱狀態轉成權限。

  • 免費空間、私密空間、邀請制空間
  • 付費會員、試用權限、VIP tier
  • 課程 access、cohort access
  • Role-based access
  • 課程、聊天室、資源、活動解鎖
Module 6

內容與公告 / Content & Announcements

必要

Purpose：讓空間主用低噪音方式發布官方內容、公告、活動和服務更新。

  • 公告、內容貼文、活動通知
  • 商品 / 服務卡、交付內容更新
  • 置頂、保存、轉發
  • 導向討論區、私聊或付費入口
  • 發布權限與審核紀錄
Module 7

討論區 / Discussion

Purpose：讓有價值的討論可以被搜尋和沉澱，而不是被群聊速度沖走。

  • 貼文、問題、投票、公告、資源
  • wins / progress updates
  • 留言、回覆、反應
  • 最佳答案、分類、標籤
  • 置頂、收藏、檢舉、管理
  • 依最新、熱門、未回答排序
Module 8

Delivery Hub / 交付中心

可插拔活動模組

Purpose：交付課程、會員內容、文件、工具、案例、服務說明、SOP、培訓資料與直播回放。

  • 課程、會員內容、案例、服務說明
  • 影片、文字、文件、工具、連結、回放
  • 測驗、作業、完成狀態或服務交付狀態
  • 進度追蹤、交付項目留言
  • 關聯討論 topic
  • 依會員或等級解鎖
  • Resource library
Module 9

即時通訊 / Chat

Purpose：保留核心 IM 能力，形成比論壇型產品更強的關係優勢。

  • 1:1 私聊、成員互聊、空間主 / 管理員對成員聊天
  • 群聊、cohort chat、VIP chat、event chat、course discussion chat
  • 文字、語音、圖片、文件、連結預覽
  • 回覆、反應、已讀、置頂、通知
  • Contextual DM：來源內容、加入的交付項目、提問、會員狀態、近期活動
Module 10

活動 / Events

可插拔活動模組

Purpose：在空間內建立節奏和週期性參與。

  • 直播課、AMA、office hour、webinar
  • Challenge、meetup、event calendar
  • RSVP、提醒、live chat
  • 回放、出席追蹤、活動後討論
Module 11

激勵機制 / Incentives

Purpose：連接熟悉的紅包、積分、獎勵、裂變和參與激勵能力。

  • 積分、等級、徽章、排行榜
  • 每日簽到、交付項目完成獎勵、有幫助回答獎勵
  • 邀請獎勵、推薦獎勵、紅包活動
  • 群組活動獎勵、活動參與獎勵
  • 依等級解鎖內容或聊天權限
  • 反垃圾規則，避免低品質刷分
Module 12

經營後台 / Admin Console

Purpose：讓空間主把空間當成生意、服務或組織來經營。

  • 成員列表、成員活動、成員標籤
  • 批准、移除、封禁成員
  • 角色分配、內容管理、交付中心管理
  • 活動管理、公告發布、聊天管理
  • 活動分析、交付完成分析、member journey
  • 收入 / 訂閱總覽、campaign source tracking
Module 13

信任與治理 / Trust & Governance

Purpose：讓空間成長後仍安全、可管理。

  • 檢舉、封鎖、靜音、封禁
  • 管理員、版主、加入審核
  • 貼文管理、留言管理
  • 垃圾訊息偵測、連結安全
  • 付款爭議紀錄
  • 管理員操作、內容移除、權限變更紀錄
Module 14

In-space Search / 空間內搜尋

必要

Purpose：幫成員在已加入的空間內找到人、內容、討論、交付內容、活動與聊天上下文。

  • 搜尋成員
  • 搜尋公告
  • 搜尋 topic / 問答
  • 搜尋交付內容
  • 搜尋文件 / 工具 / 回放
  • 搜尋活動
  • 搜尋聊天中的上下文
Module 15

Notifications / 通知

必要

Purpose：把公告、回覆、私聊、付款狀態、活動提醒和交付進度帶回原上下文。

  • 未讀、push、inbox item
  • 公告、回覆、私聊提醒
  • 活動提醒、付款與權限變更
  • 靜音、退訂、通知強度控制
  • 回到原內容、討論或聊天
Module 16

全站空間發現 / Space Discovery

後續擴展

Purpose：幫使用者在平台層找到更多公開空間、空間主和熱門空間。

  • 搜尋公開空間
  • 按分類探索 spaces
  • 公開空間索引
  • 推薦與排序
  • 熱門空間
  • 風險審核與公開可見性
  • 私密空間與私聊內容不進全站發現
Module 17

付費與訂閱 / Payments & Subscription

必要

Purpose：支撐空間主變現和付費空間，但不變成標準電商平台。

  • 月訂閱、年訂閱、一次性交付內容付款
  • 付費 cohort、VIP 會員、coupon、trial
  • 退款狀態、付款紀錄、訂閱狀態
  • 與 access control 整合
